New Profiling Point Wizard: Reset Results

Use to reset the collected profiling results each time that a thread in the application hits the profiling point.

Element Description

Name

 Specifies the name that you want to use to identify the profiling point.

This name is used to identify the profiling point in the Profiling Points window.

Location

 Specifies the name of the file and the line in the source code where you want to place the profiling point. Click Browse to locate the file in your project.

Select Begin or End to specify if the profiling point is hit when a thread hits the beginning of the specified line or reaches the end of the line.
