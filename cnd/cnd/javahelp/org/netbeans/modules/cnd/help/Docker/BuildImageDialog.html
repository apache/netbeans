Building a Docker Image

See Also

A Dockerfile is a text document that contains a set of instructions which is used for building an image.


To Build a Docker Image

Before building an image you can setup a Docker Instance, see Adding a Docker Instance.


  1. In the Projects window, right click on Dockerfile and click Build to open the Build Image wizard.
  2. In the Build Image wizard, select Build Instance or click on the Add option.

    3. A Build Instance is a host machine on which build action will be executed.

  3. Click Next and on the Build Context step, fill in the following fields:
    • Build Context: A directory which is uploaded with all the contents to the Build Instance host.
    • Repository [Optional]: The set of Docker images that can be shared by pushing it to a registry server. The different images in the repository can be labelled using tags.
    • Tag [Optional]: The label applied to the Docker image in a repository. Using Tag, various images can be distinguished.
  4. Click Next and on the Build Options step, fill the following:
    • Dockerfile: Select the relative path to the Dockerfile.
    • Build-time variables: Specify the values for the several Build-time variables.
  5. Click Finish.

    6. After you click Finish, the new Docker Image is built. It appears in the Images tab of the corresponding Build Instance.
