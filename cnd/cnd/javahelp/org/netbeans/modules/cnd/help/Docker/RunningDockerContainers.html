Running Docker Containers

See Also

A container is a runtime instance of a Docker image. A Docker container consists of the following components:

  • A Docker image
  • An execution environment
  • A standard set of instructions

To Run Docker Containers:

  1. In the services window, right click on the image you want to run and select Run.
  2. In Container Properties, fill in the Name, Command, and User fields.
  3. Select the options that you want to use and then click Next.
    • Keep STDIN open even if not attached (-i, --interactive)

      When set to true, keep stdin open even if not attached.

    • Allocate a pseudo-TTY (-t or --tty flag)

      When set to true, Docker can allocate a pseudo-tty and attach to the standard input of any container. For example, this can be used to run a throwaway interactive shell. The default is false.

    • Run in privileged mode (--privileged)

      Gives extended privileges to this container. A privileged container is given access to all devices.

      By default, Docker containers are unprivileged and cannot run a Docker daemon inside the Docker container. This is because, by default, a container is not allowed to access any devices.

    • Mount Volumes (-v or --volume)

      Creates a bind mount. When you specify the Source and Target directories, Docker bind mounts Source directory in the host to Target directory in the Docker container.

  4. Specify the Port Bindings property during the Run Image action. You can view the port bindings by selecting View Port Bindings on the running container.

For more information on Port Bindings, see Bind container ports to the host on the Docker Documentation website.

The mount volume table enables you to specify the directory on the host Docker machine that you want to mount into a container.
