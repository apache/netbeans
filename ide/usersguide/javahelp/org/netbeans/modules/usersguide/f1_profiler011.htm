New Profiling Point Wizard: Triggered Take Snapshot

Use to take profiling snapshots or heap dumps when specific conditions are met.

Element Description

Name

 Specifies the name that you want to use to identify the profiling point.

This name is used to identify the profiling point in the Profiling Points window.

Settings

 Snapshot options

Take

 Specifies the type of snapshot that you want to take.

Save

 Specifies the location for the saved snapshots. You can save snapshots to the default directory in your project or you can specify a location on your local system.

Reset results after taking snapshot

 Resets the collected profiling results after each snapshot is taken.

Time

 Specifies the conditions that trigger the IDE to take the snapshot. Select a condition from the Take When drop-down list and select a value according to the type of condition that you select.

If you select Take Once, the IDE only takes the snapshot the first time that the selected condition is met.

If you select Take Every, the IDE takes a snapshot each time that the selected condition is met.
